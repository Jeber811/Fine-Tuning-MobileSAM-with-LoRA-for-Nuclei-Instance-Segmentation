# Fine-Tuning-MobileSAM-with-LoRA-for-Nuclei-Instance-Segmentation

This repository contains the implementation for fine-tuning the Mobile Segment Anything Model (MobileSAM) specifically for nuclei instance segmentation in H&E-stained histological images. The fine-tuning process utilizes a Parameter-Efficient Fine-Tuning (PEFT) strategy via Low-Rank Adaptation (LoRA).

# Directory Structure

The project relies on two primary directories (to be uploaded):
* NuInsSeg/: This directory contains the base dataset (images and ground truth masks). The tissue images are the only images needed from the dataset for instance segmentation.
* all_results/: This directory contains the output segmentation masks generated by the model. The label masks modify masks are recommended for instance segmentation.

# Implementation & Environment Details
This project was initially developed and executed in a Kaggle notebook environment. However, it is designed to be portable and can be run locally or on any other cloud platform.

To run outside of Kaggle: You will need to modify the input paths in the code to point to your local NuInsSeg and all_results directories.

# Hardware Recommendations
Because this implementation utilizes MobileSAM (a lightweight TinyViT image encoder) combined with parameter-efficient LoRA, it is highly optimized for lower-resource environments.

* Recommended GPU: A single mid-range GPU with at least 8GB of VRAM (e.g., NVIDIA RTX 3060, Kaggle's NVIDIA T4 x2, or P100) is more than sufficient for training and inference.

# Model & LoRA Parameters

LoRA matrices were injected specifically into the Attention blocks of the TinyViT Image Encoder (wrapping the Query and Value linear projection layers), while freezing the massive original base model weights.

* LoRA Rank ($r$): 4
* LoRA Alpha ($\alpha$): 8
* Total Tunable LoRA Parameters Added: 29,696

# Training & Accuracy Improvements

The current baseline for this implementation was trained across 10 epochs using a 5-Fold Cross Validation approach.

How to Improve Accuracy: Training was stopped early at 10 epochs to save on computation time and costs. During testing, the quantitative evaluation metrics (Semantic Dice Score, Aggregated Jaccard Index, and Panoptic Quality) were still showing an upward trend. You can improve the model's accuracy and potentially outperform baseline methods simply by increasing the total number of epochs in the training configuration.
